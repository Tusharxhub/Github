# Issues

## What is it?
Issues are a bug-tracking and task-management feature built into GitHub. They are basically a forum thread attached to a repository where people can discuss things that need to be done.

## Why do we use it?
If you find a bug in a project, want to suggest a new feature, or need to track your own to-do list for a repository, you create an Issue. It keeps all project management organized right next to the code.

## Basic Syntax
Issues are managed entirely on the GitHub website.

## Example
1. Go to a repository on GitHub.
2. Click the **Issues** tab near the top.
3. Click the green **New issue** button.
4. Give it a descriptive title (e.g., "Login button is misaligned on mobile").
5. Write a detailed description of the problem, including steps to reproduce it.
6. Click **Submit new issue**.

## Common Mistakes
- **Vague titles:** "It's broken" is a terrible issue title. "Crash when clicking submit on checkout page" is a great title.
- **No reproduction steps:** If you report a bug, tell the developers exactly how to trigger the bug so they can fix it.

## Quick Summary
- Used for tracking bugs, tasks, and feature requests.
- Managed on the GitHub website.
- Always provide clear details and steps to reproduce bugs.

# Discussions

## What is it?
GitHub Discussions is a collaborative communication forum for a repository. It's like a Reddit or StackOverflow specifically for that one project.

## Why do we use it?
Issues are meant for actionable tasks (like bugs or specific features). Discussions are meant for open-ended conversations, Q&A, sharing ideas, or showing off what you built with the project. It prevents the Issue tracker from getting cluttered with questions.

## Basic Syntax
Discussions are created and managed on the GitHub website.

## Example
1. Go to a repository that has Discussions enabled (like Next.js or React).
2. Click the **Discussions** tab.
3. You can browse categories like "Q&A", "Show and tell", or "General".
4. Click **New discussion** to ask a question or start a conversation.

## Common Mistakes
- **Putting bugs in Discussions:** If something is broken in the code, open an Issue.
- **Putting questions in Issues:** If you just need help understanding how to use the software, open a Discussion.

## Quick Summary
- A forum for open-ended conversations and Q&A.
- Keeps the Issue tracker clean for actual bugs.
- Found on the GitHub website (if the repo owner enabled it).

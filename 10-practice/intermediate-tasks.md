# Intermediate Tasks

Ready for a challenge? Try these intermediate tasks to practice branching and merging.

## Tasks

- [ ] **1. Branch:** In your existing practice repository, create a new branch called `feature-style`.
- [ ] **2. Switch:** Switch to the `feature-style` branch.
- [ ] **3. Edit:** Add some CSS code or extra text to one of your files.
- [ ] **4. Commit:** Stage and commit this change on the `feature-style` branch.
- [ ] **5. Switch Back:** Switch back to the `main` branch. (Notice how your edits disappear? They are safely stored on the other branch!)
- [ ] **6. Merge:** While on the `main` branch, run `git merge feature-style`.
- [ ] **7. Push:** Push the updated `main` branch to GitHub.
- [ ] **8. Conflict Practice:** 
  - Create two branches: `branch-a` and `branch-b`.
  - Edit the exact same line of the exact same file in both branches.
  - Commit the changes on both branches.
  - Switch to `main` and merge `branch-a`.
  - Now, try to merge `branch-b` into `main`. You will get a merge conflict!
  - Open the file, resolve the conflict, and complete the merge.

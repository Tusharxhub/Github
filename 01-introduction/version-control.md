# Version Control

## What is it?
Version control is a system that records changes to a file or set of files over time so that you can recall specific versions later.

## Why do we use it?
Imagine writing a long document. Without version control, you might end up with files like `project_v1`, `project_v2_final`, `project_v3_really_final`. Version control eliminates this mess. It allows you to revert files back to a previous state, revert the entire project back to a previous state, compare changes over time, see who last modified something that might be causing a problem, and who introduced an issue and when.

## Quick Summary
- Tracks changes to files over time.
- Acts like a "Time Machine" for your code.
- Allows multiple people to work on the same project without overwriting each other's work.
- Helps identify who made a change and why.

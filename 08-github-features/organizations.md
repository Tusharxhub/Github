# Organizations

## What is it?
A GitHub Organization is a shared account where multiple users can collaborate across many repositories simultaneously, with sophisticated permission levels.

## Why do we use it?
If you are running a company or a large open-source project, having repositories tied to your personal account (`github.com/tushar/project`) is bad for scaling. An organization (`github.com/company-name/project`) allows you to create teams (e.g., "Frontend Developers", "Managers"), assign different access rights, and transfer ownership easily.

## Basic Syntax
Organizations are created and managed entirely on the GitHub website.

## Example
1. Go to your GitHub profile menu and click **Your organizations**.
2. Click **New organization**.
3. Choose the Free plan (for most open-source or small teams).
4. Name the organization (e.g., `My-Startup-Name`).
5. Invite your team members.
6. Create repositories inside the organization.

## Common Mistakes
- **Using a personal account for a company:** Always create an Organization for company code so that if the founder leaves or loses their account, the company doesn't lose access to its codebase.

## Quick Summary
- A shared workspace for teams and companies.
- Provides fine-grained access control (Admin, Write, Read).
- Keeps professional code separate from personal projects.

# README.md

## What is it?
A `README.md` is a text file written in Markdown format that lives in the root of your project folder. GitHub automatically displays the contents of this file on the front page of your repository.

## Why do we use it?
It acts as the welcoming page and instruction manual for your project. Without a good README, people visiting your repository won't know what your code does, how to install it, or how to use it.

## Basic Syntax
Markdown is used to format the text.
```md
# Project Title
A short description of the project.

## How to Install
Run this command:
`npm install`
```

## Example
A professional README should include:
1. **Title and Description:** What is this project?
2. **Installation Instructions:** Step-by-step how to get it running locally.
3. **Usage:** How to use the software.
4. **Tech Stack:** Languages and frameworks used.
5. **License:** How others are allowed to use your code.

## Common Mistakes
- **Leaving it blank:** An empty README makes your project look unfinished or abandoned.
- **Putting private info:** Never put API keys, passwords, or personal data in a README.

## Quick Summary
